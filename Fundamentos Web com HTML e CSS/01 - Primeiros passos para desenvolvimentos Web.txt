**O que são redes?**

 

- Backbone – Estruturas que gerenciam o trafego pela internet, que interligando outras backbones. 

- Provedor de acesso – empresas que fornecem ao cliente final 

- Provedor de serviço – dial-up (discada), adsl, fibra ótica, rádio, satélite, móvel, P2P

 

 

**TCP/IP**

 

- TCP – Transmission control protocol

- IP – Internet Protocol

 

Uma comunicação passa por 4 etapas antes de começar a transmissão:

- Física (ex.: placa de rede)

- Rede (ex.: IP)

- Transporte (ex.: TCP, UDP)

- Aplicação (ex.: FTP, SMTP, http)

 

UDP x TCP

 

- UDP – rápido, não confiável, livestream

- TCP – voltado a conexão, handshake, integridade, ordem dos dados, app msg txt

 

**Portas (ports)**

 

As portas por onde os dados sairão e chegarão

 

- 20: FTP

- 22: SSH

- 25: SMTP

- 53: DNV

- 80: http

- 443: HTTPS

**Roteador**

**Switch**

**Dados Móveis**

 

- 1G (analógico): 10 kbps

- 2G (digital) 97 kbps

- GPRS / 2,5G: 32-80 kbps

- EDGE: 2,75G 128-236 kbps

- 3G: 7 Mbps / 4G: 22,1 Mbps / 5G: 10 Gbps

 

**Wi-fi**

 

- 802.11: 2,4 GHz, 2Mbps

- 802.11a: 5 GHz, 54Mbps

- 802.11b: 2,4 GHz, 11Mbps

- 802.11g: 2,4 GHz, 54 Mbps

- 802.11n: 2,4 GHz/5 GHz, 150 – 600 Mbps

 

- WEP – chaves de 64 e 128 bits

- WPA – chave trocada periodicamente

- WPA2 (AES) (80211i) – mais segurança e requer mais processamento

**Browser**

 

Identifica vários tipos de LP, marcação e multimídia. Existem plug-ins ou add-nos que ajudam. Cachê e cookies.

Página estática – quase nenhuma interação, apenas te leva a outro site

Página dinâmica – barras de rolagem, vídeos que correm, zoom, coisas independentes.

 

**Sites**

 

Página de internet, diversos propósitos, usadas diversas LP.

 

**App**

 

Software executado no navegador, app cellular.

 

**Webserver**

 

- Estático – servidos físico onde são armazenados arquivos, softwares e/ou banco de dados.

- Dinâmico – O dinâmico se refere aos softwares que estão presentes no servidor físico.

- Arquivos (file Server), Aplicações (application Server), Banco de dados (database).

- WebService – Interface disponível para fazer requisições e consultas em bancos de dados inacessíveis. (Governo, correios)

 

**Stacks**

 

Conjunto de softwares necessários e suficientes para executar um app/programa. LP. Ambientes e ferramentas de interação com o app/sw. Capacidade e limitação de performance. Pontos fortes e fracos do app/sw.

 

**Front-end**

 

Parte da frente, interface dos sites, software, app, WebService. Interface (UI, UX), Lógica de programação, HTML, CSS. Jquery/JS/AJAX, PHP, Bootstrap, outros frameworks.

 

**Back-end**

 

Parte de trás, servidores, banco de dados, entre interface e banco de dados, validações, MySQL, Oracle, protocolos de comunicação, PHP, Java, node.js

 

**Fullstack**

 

Profissional que trabalha em todas as camadas das tecnologias de desenvolvimento/execução.

 

**Principais LPs**

 

HTML, CSS, JavaScript (jQuery, AJAX, diversas libs), PHP, .NET, ASP, Java, Ruby, Python, Perl, C/C++/C#
